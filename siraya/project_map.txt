siraya/services/llm_service.py  —  LLM Service (The State Machine)
================================================================================
  Responsabilità : Gestore AUTONOMO della conversazione.  Il metodo pubblico
                   generate_response(user_input, session_state) incapsula
                   l'intera logica decisionale:
                   INTAKE → CLINICAL_TRIAGE → RECOMMENDATION.
  Classi / funzioni:
    - SymptomNormalizer          → fuzzy-matching sintomi (medicalizzazione)
    - DiagnosisSanitizer         → blocco diagnosi/prescrizioni vietate
    - LLMService
        ── generate_response()   → ★ ENTRY POINT: state-machine autonoma
        ── _determine_phase()    → rileva INTAKE / CLINICAL / RECOMMENDATION / INFO
        ── _extract_inline_data()→ estrae età, sesso, località, dolore dall'input
        ── _phase_intake()       → fase 1: accoglienza, dati base, ZERO RAG
        ── _phase_clinical()     → fase 2: triage clinico con Lazy RAG
        ── _phase_recommendation()→ fase 3: SBAR + ricerca struttura
        ── _phase_info()         → branch INFO: interroga master_kb.json
        ── _call_llm()           → chiamata API low-level (Groq → Gemini fallback)
        ── _parse_options()      → estrae opzioni A/B/C dalla risposta LLM
        ── _get_conversation_ctx()→ formatta ultimi messaggi per il prompt
        ── _log_interaction()    → logging Supabase
        ── check_emergency()     → keyword emergenze critiche
        ── get_ai_response()     → metodo legacy (chiama _call_llm internamente)
        ── _build_intake_prompt / _build_system_prompt_with_rag /
           _build_recommendation_prompt → prompt-builder per ogni fase
    - get_llm_service()  → singleton

siraya/services/rag_service.py  —  RAG Service (Lazy RAG)
================================================================================
  Responsabilità : Retrieval-Augmented Generation tramite full-text search
                   su Supabase.  Attivato SOLO nelle fasi cliniche esplicite
                   (FASE_4_TRIAGE, FAST_TRIAGE_A, VALUTAZIONE_RISCHIO_B).
  Classi / funzioni:
    - RAGService
        ── should_use_rag(phase, msg)   → gatekeeper (Lazy RAG)
        ── retrieve_context(query, k)   → cerca chunks
        ── format_context_for_llm()     → formatta per prompt
        ── get_stats()                  → statistiche DB
    - get_rag_service()  → singleton (st.cache_resource)
